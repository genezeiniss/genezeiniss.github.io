ðŸ“Œ Case Study: Implementing a Hybrid SDK for Sensitive Data Detection

Background

At Intuit, as the lead engineer for a Java-based application that detects sensitive data, I was responsible for preventing data leakage and safeguarding customer information. The application included a RESTful service and a Hybrid SDK designed for latency-sensitive clients.

Challenge

Frequent Updates: The classifiers used for sensitive data detection evolved continuously, requiring a way to push updates without forcing SDK upgrades.

Solution

To overcome these challenges, I designed and developed a Hybrid SDK architecture that decoupled dynamic components, such as data classifiers and validation logic, from the SDK itself. This allowed us to:

  • Offload dynamic configuration to an external repository.
  • Load classifiers dynamically without requiring SDK updates.
  • Monitor SDK usage and performance across different clients.

Results & Impact

  • Eliminated Forced SDK Upgrades: Clients could seamlessly receive classification updates without modifying their SDK version.
  • Improved Performance: The Hybrid SDK optimized latency by combining local execution with dynamic remote configuration.
  • Better Visibility & Control: The monitoring system provided insights into SDK usage, client versions, and performance trends.
  • Scalability for Future Growth: The architecture supported Intuitâ€™s roadmap to expand classification coverage from 40 to over 200 data types.
